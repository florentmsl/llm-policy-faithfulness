# Task Description

Environment:
Skiing (Atari 2600). The player skis down a slope, steering left or right to pass between gates (pairs of flags).

Objects and dynamics:
- Player1 skis continuously downhill; the slope (terrain) scrolls upward on screen.
- Flag1..FlagN mark the gates that must be passed between. Missing a gate adds time penalty.
- Tree1..TreeN are obstacles; collisions slow the skier.
- The clock runs; the episode ends when the player reaches the finish line. Lower time is better.

Controls (action codes):
- 0 NOOP  (ski straight)
- 1 RIGHT (steer right)
- 2 LEFT  (steer left)

Note: in this game RIGHT/LEFT steer the skier; there is no UP/DOWN. The skier always descends; its downhill speed is modulated by how sharply it is turning.

Relevant observed objects (from the SCoBots feature set):
Player1, Flag1..Flag8, Tree1..TreeN.

Coordinate convention: y increases downward; the skier's vertical position on screen is roughly constant as the world scrolls.

An episode ends when the player crosses the finish line.

Task objective:
Ski to the finish line as quickly as possible, steering between each pair of flags and avoiding trees.

Reward function used during training:
```python
def reward_fn(prev_state, state):
    """
    Representative Skiing-style reward:
    Small negative reward each step (time penalty), large bonus when a gate
    is passed correctly, and a large positive reward when the finish line is
    reached. Missing a gate adds an extra time penalty.
    """
    r = -0.01  # constant time penalty per step
    if state["gates_passed"] > prev_state["gates_passed"]:
        r += 1.0
    if state["finished"] and not prev_state["finished"]:
        r += 10.0
    if state["gates_missed"] > prev_state["gates_missed"]:
        r -= 1.0
    return r
```

# Example (only for reference)
Here is an example of how to analyze and explain a symbolic policy (only for reference):
Policy:
```python
def play(state):
    if state["D(Ball, Paddle).y"] > 2:
        return 2  # DOWN
    if state["D(Ball, Paddle).y"] < -2:
        return 1  # UP
    return 0  # NOOP
```

Explanation:
This policy makes the paddle follow the ball's vertical position. When the ball is lower than the paddle, it moves the paddle down; when the ball is higher, it moves the paddle up; and when they are already close in height, it waits. Overall, it behaves like a simple vertical tracking controller.
